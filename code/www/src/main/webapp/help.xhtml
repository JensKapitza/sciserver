Such Beispiele
Das % Zeichen hat eine erweiterte Funktion, und hilft bei der Suche. Beispiele dafür hier zu finden.
Tags und Pfad leer
Sind Tags und Pfad leer werden alle Dateien angezeigt.
Tags: origin=
Suche nach allen Tags die mit origin starten und beliebigen Wert haben. z.b. origin=krefeld
Tags: =krefeld
Suche nach allen Tags die als Wert krefeld haben, der Schlüssel kann beliebig sein. z.b. origin=krefeld
Tags: origin%
Suche nach allen Tags die mit origin starten. z.b. origin=krefeld
Tags: %origin%
Suche nach allen Tags die origin enthalten. z.b. myoriginal, state=original, origin=krefeld
Tags: %true
Suche nach allen Tags die auf origin enden. z.b. gpl=true
Pfadteil: %musik.mp3
Suche nach Pfad der auf musik.mp3 endet.
Pfadteil: %musik%
Suche nach Pfad der musik enthält.
Pfadteil: %musik%peter%mp3
Suche nach Pfad der musik und peter enthält und auf mp3 endet.
Tags: home% AND windows
Tags und Attribute können mit AND verknüpft werden, z.b. alle windows getaggten Dateien die auch mind. home als Tag/Attribute enthalten.
Befehle Beispiele
Sollen auf dem provider Befehle der Art:
befehl.path=PATH
befehl.replacewithpath=REGEX
konfiguriert sein, so kann dieser unter RUN durch "befehl +argument1 +argument2 ..." verwendet werden. Dabei wird der befehl zu dem unter PATH angegebenen Programm expandiert und mit den argumenten, die entsprechend dem REGEX gefiltert und mit dem Pfad der Datei versehen werden, aufgerufen.
